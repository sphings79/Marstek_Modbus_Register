## Deutsch

### Überblick

Dieses Dokument enthält die Ergebnisse des Modbus-TCP-Scans des Registerblocks 40000–49999 eines **Marstek Venus D** Heimspeichers. Er ergänzt den [Scan des 30000-Blocks](./marstek_venus_d_register_analysis.md).

**Rahmenbedingungen:**
- Gerät: Marstek Venus D mit 2 Batterie-Packs à 16 Zellen (LFP)
- Verbindung: Modbus TCP, IP `192.168.181.154`, Port `502`, Unit ID `1`
- Gescannte Register: 40000–49999 (10.000 Stück)
- Ergebnis: **93 Register mit gültigen Daten**, 9.907 leer/stumm

Der 40000-Block enthält hauptsächlich **Steuer- und Konfigurationsregister** — im Gegensatz zum 30000-Block, der Messwerte liefert. Schreibzugriffe auf Register im Bereich 42000–42999 sind nur möglich, wenn das Gerät im **RS485-Steuermodus** ist (42000 = 0x55AA).

> ⚠️ **Achtung:** Register im 42000er-Bereich sind **schreibbar** und können das Geräteverhalten direkt beeinflussen (erzwungenes Laden/Entladen, Leistungslimits). Niemals Werte schreiben, ohne das Gerät vollständig zu verstehen!

Die Zuordnung erfolgte anhand der README des [ViperRNMC/marstek_venus_modbus](https://github.com/ViperRNMC/marstek_venus_modbus)-Projekts mit folgender Priorität:

| Priorität | Spalte | Begründung |
|:---------:|:------:|:-----------|
| 1 | `d` | Venus D – primäre Quelle |
| 2 | `e_v3` | Fallback wenn `d` leer |
| 3 | `a` | Letzter Ausweg |
| ❌ | `e_v12` | **Ausgeschlossen** |

### Hinweise zu den String-Blöcken 41500–41515

Die Register 41500–41515 enthalten eine Mischung aus lesbaren ASCII-Strings und binären Werten. Die wahrscheinlichste Interpretation:

| Reg.-Bereich | Dekodiert | Vermutung |
|:------------:|:---------:|:----------|
| 41500–41503 | `SM471` | Modell-ID oder SN-Präfix des Kommunikationsmoduls |
| 41504–41509 | `Hame147258` | WiFi-SSID des eingebauten Hotspots |
| 41510–41511 | `0xE94B / 0x0120` | WiFi-Passwort (binary) oder Verschlüsselungsparameter |
| 41512–41515 | `Write ee` | Bedeutung unklar — möglicherweise eine Firmware-interne Bezeichnung |

> ⚠️ Wenn `Hame147258` tatsächlich die SSID des WLAN-Moduls ist, handelt es sich um eine potenzielle **Sicherheitsinformation**. Es empfiehlt sich, das Gerät in einem dedizierten IoT-VLAN zu betreiben und den Direktzugriff auf die Kommunikationsschnittstelle zu prüfen.

## English

### Overview

This document contains the results of the Modbus TCP scan of the 40000–49999 register block on a **Marstek Venus D** home battery system. It supplements the [30000 block scan](./marstek_venus_d_register_analysis.md).

**Test setup:**
- Device: Marstek Venus D, 2 battery packs × 16 cells (LFP)
- Connection: Modbus TCP, IP `192.168.181.154`, port `502`, Unit ID `1`
- Scanned registers: 40000–49999 (10,000 total)
- Result: **93 registers with valid data**, 9,907 empty/silent

The 40000 block contains primarily **control and configuration registers**, as opposed to the 30000 block which holds measurement values. Write access to registers in the 42000–42999 range is only possible when the device is in **RS485 control mode** (42000 = 0x55AA).

> ⚠️ **Warning:** Registers in the 42000 range are **writable** and can directly influence device behavior (forced charge/discharge, power limits). Never write values without fully understanding the device!

Registers 41500–41515 contain a mix of readable ASCII strings and binary values. The most likely interpretation:

| Register range | Decoded | Interpretation |
|:--------------:|:-------:|:---------------|
| 41500–41503 | `SM471` | Model ID or SN prefix of the comm module |
| 41504–41509 | `Hame147258` | WiFi SSID of the built-in hotspot |
| 41510–41511 | `0xE94B / 0x0120` | WiFi password (binary) or encryption parameters |
| 41512–41515 | `Write ee` | Meaning unclear — possibly a firmware-internal label |

> ⚠️ If `Hame147258` is indeed the WiFi SSID of the comm module, this constitutes **potentially sensitive network information**. It is recommended to operate the device in a dedicated IoT VLAN and to review direct access to the communication interface.

## Scan Setup

```bash
python modbus_scanner.py \
  --ip 192.168.181.154 \
  --port 502 \
  --start 40000 \
  --count 10000 \
  -v \
  --heartbeat-reg 41000 \
  --csv scan_40000.csv
```

| Parameter | Value |
|:----------|:------|
| Delay between registers | 0.3 s |
| Timeout per request | 3.0 s |
| Pause interval | 3.0 s every 50 registers |
| Backoff | max. 5 retries → 30 s cooldown |
| Scan mode | single pass (no loop) |
| Registers scanned | 10,000 (40000–49999) |
| Registers with data | 93 |
| Errors | 0 |
