## Deutsch

### Überblick

Dieses Dokument enthält die Ergebnisse eines vollständigen Modbus-TCP-Scans des Registerblocks 30000–39999 eines **Marstek Venus D** Heimspeichers. Der Scan wurde mit einem eigenen Python-Skript (`modbus_scanner.py`) durchgeführt.

**Rahmenbedingungen:**
- Gerät: Marstek Venus D mit **2 Batterie-Packs à 16 Zellen** (LFP)
- Verbindung: Modbus TCP, IP `192.168.181.154`, Port `502`, Unit ID `1`
- Gescannte Register: 30000–39999 (10.000 Stück)
- Ergebnis: **278 Register mit gültigen Daten**, 1 Fehler (Reg. 32085), Rest leer/stumm

Die Zuordnung erfolgte anhand der README des [ViperRNMC/marstek_venus_modbus](https://github.com/ViperRNMC/marstek_venus_modbus)-Projekts mit folgender Priorität:

| Priorität | Spalte | Begründung |
|:---------:|:------:|:-----------|
| 1 | `d` | Venus D – primäre Quelle |
| 2 | `e_v3` | Fallback wenn `d` leer |
| 3 | `a` | Letzter Ausweg |
| ❌ | `e_v12` | **Ausgeschlossen** – komplett anderes Registerlayout |

> **Hinweis zu Battery Pack 2:** Die README definiert für das Modell D nur Pack-1-Register (ab 34000). Im Scan zeigt sich jedoch eindeutig, dass Pack 2 exakt **+100 Register** später beginnt (34100) und die Blockstruktur von Pack 1 spiegelbildlich wiederholt. Diese Einträge sind als `d (inferred)` gekennzeichnet.

> **Hinweis zu Zellanzahl:** Das Venus-D-Modell verwendet **16 Zellen** pro Pack (Register 34018–34033). Das A-Modell hat nur 13 Zellen. Der Ev12 hat ein vollständig abweichendes Layout und wurde nicht berücksichtigt.

## English

### Overview

This document contains the results of a full Modbus TCP scan of the 30000–39999 register block on a **Marstek Venus D** home battery system. The scan was performed using a custom Python script (`modbus_scanner.py`).

**Test setup:**
- Device: Marstek Venus D with **2 battery packs × 16 cells** (LFP)
- Connection: Modbus TCP, IP `192.168.181.154`, port `502`, Unit ID `1`
- Scanned registers: 30000–39999 (10,000 total)
- Result: **278 registers with valid data**, 1 error (reg. 32085), rest empty/silent

Registers were matched against the README of the [ViperRNMC/marstek_venus_modbus](https://github.com/ViperRNMC/marstek_venus_modbus) integration using the following priority order:

| Priority | Column | Reason |
|:--------:|:------:|:-------|
| 1 | `d` | Venus D — primary source |
| 2 | `e_v3` | Fallback if `d` is blank |
| 3 | `a` | Last resort |
| ❌ | `e_v12` | **Excluded** — completely different register layout |

> **Note on Battery Pack 2:** The README only defines Pack 1 registers for the D model (starting at 34000). The scan clearly shows that Pack 2 begins exactly **+100 registers** later (at 34100), mirroring the Pack 1 block structure. These entries are marked `d (inferred)`.

> **Note on cell count:** The Venus D uses **16 cells per pack** (registers 34018–34033). The A model has only 13 cells. The Ev12 has a completely different layout and was intentionally excluded.
